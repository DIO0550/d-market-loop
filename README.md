# d-market-loop

Claude Code プラグインで、タスクベースの開発ワークフローを自動化します。タスクファイルを順番に処理し、**実装 → コミット → PR作成 → レビュー → 修正 → マージ** のサイクルを自律的にループ実行します。

## 概要

開発タスクをMarkdownファイルとして定義すると、Claude がそれを1つずつ取り出して実装し、PRの作成からレビュー対応・マージまでを自動で行います。外部シェルスクリプト (`run-loop.sh`) がClaude のセッションを管理し、レビュー待ちのポーリングを担当することで、長時間の自律的なタスク処理を実現します。

## 主な特徴

- **タスクの自動実行**: `tasks/todo/` に配置したタスクファイルをファイル名順に処理
- **完全な開発サイクル**: ブランチ作成 → 実装 → テスト → コミット → PR → レビュー対応 → マージ
- **レビュー対応**: レビューコメントを分析し、修正が必要なら自動で対応（最大反復回数設定可）
- **エラーリカバリ**: 失敗時の状態記録と復旧処理
- **セッション再開**: 中断からの再開に対応（`task-loop-state.json` で状態管理）
- **コマンドホワイトリスト**: 許可コマンドの制御によるセキュリティ確保
- **コマンドインジェクション防止**: `&&`, `||`, `;`, `|` 等のチェーン検出・ブロック

## プラグイン構成

```
plugins/task-loop/
├── plugin.json              # プラグインメタデータ
├── hooks/                   # ライフサイクルフック
│   ├── hooks.json
│   ├── post-compact-hook.sh # コンパクト時の状態復元
│   └── session-export-hook.sh # セッション終了時のログ出力
└── skills/
    ├── task-loop-setup/     # 初期セットアップ
    ├── task-loop-run/       # タスク実行
    └── task-loop-doc/       # タスクファイル・計画書生成
```

## スキル

### task-loop-setup

プロジェクトに task-loop を導入するための初期セットアップを行います。

- 設定ファイル (`task-loop-config.json`) の生成
- タスクディレクトリ (`tasks/todo/`, `processing/`, `done/`, `failed/`) の作成
- 実行スクリプト (`run-loop.sh`) の配置
- コマンドホワイトリスト用 PreToolUse フックの生成（オプション）

### task-loop-run

タスクを1つずつ処理するメイン実行スキルです。3つのモードで動作します。

| モード | 内容 |
|--------|------|
| `implement` | タスク初期化 → 実装 → コミット → PR作成 |
| `review-check` | レビュー分析 → 修正 or マージ → 状態更新 |
| `error` | エラーリカバリ |

### task-loop-doc

ユーザーの要件をタスクファイルに分割し、プロジェクトコンテキスト (`Task.md`) を生成します。

- 要件の直接入力、既存ドキュメント、GitHub Issue からの変換に対応
- 1タスク = 1PR = 1つの独立した変更 の原則でタスクを分割

## 使い方

### 1. セットアップ

Claude Code でプラグインをインストールした後、セットアップスキルを実行します。

```
> task-loop のセットアップをして
```

対話形式でベースブランチ、レビュアー、マージ戦略などを設定します。

### 2. タスクの作成

要件からタスクファイルを自動生成します。

```
> タスクを分割して
```

または `tasks/todo/` に手動でタスクファイルを配置します。

### 3. 実行

生成された `run-loop.sh` を実行します。

```bash
bash run-loop.sh
```

## 設定 (task-loop-config.json)

| 項目 | デフォルト | 説明 |
|------|-----------|------|
| `baseBranch` | `main` | ベースブランチ |
| `reviewer` | `copilot` | PRレビュアー |
| `mergeStrategy` | `squash` | マージ方法 (`squash` / `merge` / `rebase`) |
| `maxTasks` | `0` | 最大処理タスク数（0 = 無制限） |
| `reviewMaxWaitMinutes` | `30` | レビュー待ちタイムアウト（分） |
| `maxFixIterations` | `3` | レビュー修正の最大反復回数 |
| `allowedCommands` | `[]` | 追加許可コマンド |
